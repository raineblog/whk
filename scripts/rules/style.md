# 文档排版与讲解规范（v2）

本规范整合了排版、讲解风格、思维逻辑和知识框架四个维度的指导，旨在为文档编写提供统一、可践行的标准。

---

## 一、排版规范

### 1.1 标题层级

- **一级标题（`#`）**：整篇文档只能有一个，作为文档总标题。
- **二级标题（`##`）**：划分主要知识板块或章节，可有多个。
- **三级标题（`###`）**：划分章节内的具体知识点，是文档目录的主要层级。最多使用三级标题作为文档的目录。
- **四级标题（`####`）**：仅用于表示过于复杂的三级标题下的较小分类，不会被包含在文档目录之中。尽量避免使用。

### 1.2 空格规范

| 场景                      | 规则                 | 示例                                                |
| ------------------------- | -------------------- | --------------------------------------------------- |
| 中英文之间                | 需加空格             | 这是一个 **Lipschitz 常数** 的例子                  |
| 中文与数字之间            | 需加空格             | 2025 年、有 $1\%$ 的人                              |
| 数字和单位之间            | 无须加空格           | $\pu{1kg}$、$\pu{9.6E11}$                           |
| 度（°）和百分比与数字之间 | 不加空格             | $30°$、$50\%$                                       |
| 全角标点与其他字符之间    | 不加空格             | 「引号」后面不加空格                                |
| 分割并列关系              | 半角 `/`，前后加空格 | 输入 / 输出（前后已有空格的专有名词、全角标点除外） |
| 加粗和正文之间            | 不加空格             | 这是**加粗**文字                                    |
| 斜体和正文之间            | 除全角标点外有空格   | 这是 *italic* 文字                                  |

### 1.3 标点符号

- **中文段落**：使用全角中文标点（逗号 `，`、句号 `。`、顿号 `、`、冒号 `：`、分号 `；`、引号 `「」` 或 `""`、括号 `（）` 等）。
- **英文整句或特殊名词内部**：使用半角标点。
- **数字**：使用半角字符。
- **不允许**使用 `。。。` 代替 `……`；允许使用三个连续的英语句点 `...` 代替省略号。
- **不使用**过多重复的标点符号（如 `！！！`、`？？？`）。
- **引号方向**不要反了；简体中文可使用直角引号 `「」`，尽量不要使用英文引号。
- **加粗**和正文之间没有空格，*斜体*和正文（除全角标点外）之间有空格。
- **列表项结尾**：完整句子以句号结尾；短语不加标点。

### 1.4 数学公式格式

#### 行内公式

使用 `$...$`，嵌入正文中：

```md
函数 $f(x) = x^2$ 在区间 $[0, 1]$ 上单调递增。
```

#### 独立公式块

使用 `$$...$$`，注意 `$$` 和公式主体必须换行（不能在一行写），公式与正文之间必须有空行：

```md
根据经典不等式：

$$
E=mc^2
$$

我们得到以下结论。
```

#### 公式引导性标点

公式上一行（上一个段落）应加上引导性标点符号，例如冒号：

```md
根据能量守恒定律：

$$
E_{\text{total}} = E_k + E_p
$$
```

如果段落与公式比较连贯，可用末尾逗号；如果前面已有引号（如「提示：参考下面公式。」），则用句号。

#### 公式中的中文处理

数学公式内尽量避免使用中文。如果必须使用，采用 `<center markdown>` 标签将中文置于公式外部：

```md
<center markdown>
同化量（GPP）$=$ 净初级生产量（NPP）$+$ 呼吸消耗（R）
</center>

<center markdown>
输入 $=$ 输出（有效）$+$ 损耗（无效）
</center>
```

#### 物理量与化学式

- 物理公式中的单位使用 `\pu{}`：$\pu{1kg}$、$\pu{9.6E11}$、$\pu{E12}$。
- 化学式使用 `\ce{}`：$\ce{H2O}$、$\ce{H2 + O2 ->[点燃] H2O}$。
- 重要的化学反应方程用独立 `$$...$$` 块展示，反应条件用 `->[\text{条件}]` 标注。

#### 数值范围

使用 `$10 \sim 20$` 格式，不要使用 `10-20`、`10—20`、`10～20`、`10~20` 等：

```md
实验在 $30 \sim 50$ 的温度范围内进行。
```

#### 数字使用原则

- **使用公式**：复杂数值、带单位的数值（如 $\pu{1kg}$、$\pu{9.6E11}$）。
- **不使用公式**：年份（2000 年）、特殊意义的数字（5-羟色胺）、三四位的整数表示数量（50 光年、500 年）。
- **范围**：始终使用 `$10 \sim 20$` 格式。

#### 重要公式

重要公式可使用 `\boxed{}` 标记：

```md
$$
\boxed{E=mc^2}
$$
```

### 1.5 列表格式

- **列表项之间必须有一个空行**。
- **所有缩进必须用四个空格**，不能用 TAB。
- 列表项下的子内容用四个空格缩进。
- **尽量避免公式和列表连在一起写**。

有序列表示例：

```md
1. 第一步

    详细内容...

2. 第二步

    详细内容...

3. 第三步
```

无序列表示例：

```md
- 第一点

    详细内容...

- 第二点

    详细内容...

- 第三点
```

### 1.6 图片格式

- **必须指定图片宽度**，避免图片过大或过小：

```md
![](xxx.png){ width="100%" } ![](xxx.png){ width="80%" }
```

- 使用 `/// caption` 语法为图片添加说明文字：

```md
![](xxx.png){ width="100%" } /// caption 图 1：实验装置示意图 ///
```

- 并排图片使用 `<div class="grid" markdown>` 块：

```md
<div class="grid" markdown>

![](img1.png){ width="100%" }

![](img2.png){ width="100%" }

</div>
```

### 1.7 表格格式

- 使用标准 Markdown 语法。
- 表头居中对齐使用 `:-:`。
- 表格前后应有空行。
- 表格内容中可包含公式、图片等。

```md
|  类型  |              公式               |     说明     |
| :----: | :-----------------------------: | :----------: |
|  速度  |        $v = \frac{s}{t}$        | 匀速直线运动 |
| 加速度 | $a = \frac{\Delta v}{\Delta t}$ |   变速运动   |
```

### 1.8 补充说明与提示框

#### 提示框（默认展开）

```md
/// note | 提示标题这里是提示的内容。///
```

#### 可折叠详情（默认收起）

```md
/// details | 详细标题里面的内容默认是收起的、不展开的。///
```

#### 引用块

用于补充说明、深度拓展或标注进阶内容：

```md
> **大学物理视野**
>
> 这里是超出高中范围的拓展内容。
```

#### 卡片式内容

```md
<div class="grid cards" markdown>

- **概念名称**

    ---

    概念描述内容。

- **概念名称**

    ***

    概念描述内容。

</div>
```

### 1.9 代码块

- 使用三个反引号，必须与正文有空行。
- 如果段落是列表，则同步缩进。
- 如果代码块里面有代码块，则外面的代码块比里面的反引号多。

````md
这是一个段落。

```python
def f(x):
    return x ** 2
```

下一个段落。
````

### 1.10 分隔线

- **尽量避免使用 `---` 作为分隔线**，只在真正需要分隔不同主题时使用。

### 1.11 HTML 标签使用

- 尽量避免使用太多无用的 `<p>`、`<a>` 等 HTML 标签。
- 如果能用原生的 Markdown 语法，避免使用 HTML 标签。
- `<center markdown>` 用于含中文的公式居中展示。
- `<div class="grid" markdown>` 用于并排展示图片或卡片。

### 1.12 段落与空行

- 段落、列表、公式之间需要有足够的空行。
- 行间公式与正文之间必须有空行。
- 代码块与正文之间必须有空行。
- HTML 块与段落之间也要有空行。
- 如果段落的大部分是中文，标点尽可能使用中文全角标点。
- 不在行末以数学公式结尾（若必须，不加句号）。

## 二、讲解风格规范

### 2.1 叙述人称

- **使用第一人称「我们」**进行讲解，营造共同学习、共同探讨的氛围。
- **使用第二人称「你」**与读者对话，增加互动感和亲切感。
- **避免使用「您」**，使用「你」更亲切。
- 对自己（作者）称呼「我」或「我们」。

```md
我们先来回顾一下基本定义。你可以注意到，这里的条件非常强。
```

### 2.2 概念引入方式

引入概念时，可选择以下方式：

- **定义引入**：先给出严格定义，再展开论述。
- **问题引入**：通过一个具体问题引出概念。
- **现象/实验引入**：通过生活实例、历史事件或实验现象引入概念。
- **历史引入**：讲述概念的历史背景和发展过程。

```md
在日常生活中，我们经常遇到这样的问题：如何衡量一个函数的变化速度？这就引出了导数的概念。
```

### 2.3 语言风格

- **整体正式、准确**，类似于教科书风格。
- **在解释复杂概念时可适当口语化**，使用比喻帮助理解。
- **关键术语首次出现时同时标注中英文**：

```md
- **Lipschitz 常数**
- **MVT**（Mean Value Theorem）
- **闭区间套定理**（Nested Interval Theorem）
```

- **专有名词使用正确的大小写**。
- **对于给大众看的文章**，能用中文就避免使用英文；**对于技术性文章**，建议第一次同时指出中文和英文，之后使用习惯用法。

### 2.4 用词规范

| 规则                     | 说明                                                 |
| ------------------------ | ---------------------------------------------------- |
| 区分「的」和「地」       | 「的」修饰名词，「地」修饰动词                       |
| 避免使用「得」           | 固定用法或常用搭配除外                               |
| 不使用「一下」           | 应改为「以下」                                       |
| 不使用「带入」           | 应改为「代入」                                       |
| 注意「另」和「令」的区分 | 「另外」和「令 f(x) = 0」                            |
| 代词使用正确             | 他、她、它（以及他们、她们、它们）要用对             |
| 避免性别歧视             | 不用「他」代指女性；对同时存在多性别的群体用「他们」 |

### 2.5 例子使用

- **例子应贴近生活或具有代表性**，复杂度适中。
- **从简单到复杂递进**：先给出基本概念的例子，再给出复杂应用的例子。
- **例子应具有代表性**，能够说明概念的核心。
- **例题格式**：先题目，后解析；可提供多种解法。

```md
**例 1** 已知函数 $f(x) = x^2$，求 $f'(1)$。

**解** 由导数定义：

$$
f'(x) = 2x
$$

因此 $f'(1) = 2$。
```

### 2.6 难度递进

- **从易到难，循序渐进**。
- **基础内容与拓展内容明确分离**：拓展内容使用引用块标记，并标注「补充」「深度拓展」「前沿衔接」「大学视野」「思维拓展」等标签。
- **竞赛/大学内容明确标识**，超出高中范围但有助于理解的内容，简要提及并注明「感兴趣可以自行查询」。

### 2.7 总结与记忆

- **在关键知识点处使用口诀或总结**。
- **使用表格对比不同概念**。
- **使用决策树或流程图**总结解题方法。
- **提供做题识别要点和应试要点总结**。

```md
/// note | 应试口诀奇变偶不变，符号看象限。///
```

## 三、思维逻辑规范

### 3.1 概念引入顺序

遵循「定义 → 性质 → 应用」的基本框架：

1. **现象/问题引入**：通过生活实例、历史事件或问题引出概念。
2. **给出定义**：严格定义概念，使用数学语言。
3. **展开性质/原理**：推导或证明相关性质。
4. **实际应用**：通过例题展示概念的应用。
5. **总结归纳**：整理关键点，建立知识体系。

### 3.2 知识点之间的联系

- **在章节之间建立明确的前后呼应**。
- **同一概念在不同章节中交叉引用**。
- **跨学科联系**：建立概念之间的跨学科关联。
- **从微观到宏观的逻辑链条**。

### 3.3 类比与比喻的使用

- **善用类比将新知识与已有经验关联**，降低理解门槛。
- **用表格对比**不同概念的异同。
- **通过类比帮助记忆**。

```md
我们可以将函数的极限类比为「无限接近但不一定到达」，就像你走向一面墙，可以无限接近但永远不碰到它。
```

### 3.4 因果逻辑

- **明确标注因果关系**。
- **先说「是什么」，再说「为什么」，最后说「怎么用」**。
- **每个性质或结论后紧跟原因解释**。
- **对于存在争议或复杂机理的知识点**，应明确指出当前解释的局限性或后续拓展方向。

### 3.5 推理方式

- 使用「容易想到」、「注意到」、「不难发现」等引导性词语展示思考过程。
- 通过推导展示从已知到未知的逻辑链条。
- 提供通用方法和技巧的总结。

## 四、知识框架规范

### 4.1 文档整体结构

遵循「概念 → 性质 → 应用」的基本框架，每个二级标题下包含多个三级标题，详细讲解各个方面。

典型结构：

```md
# 文档总标题

## 概述/引言

## 基础概念

### 定义

### 性质

### 分类

## 方法/原理

### 基本方法

### 进阶方法

## 应用/实例

### 基础例题

### 综合例题

## 总结与拓展

### 小结

### 拓展内容
```

### 4.2 二级标题划分逻辑

- **按知识模块划分**，每个二级标题是一个独立的主题单元。
- **遵循由浅入深、由基础到应用的顺序**。
- 常见模式：
    - 概述/概述 → 原理/理论 → 方法/技术 → 应用/实例
    - 概念 → 性质 → 分类 → 应用
    - 历史发展 → 原理 → 操作方法 → 应用领域

### 4.3 三级标题下的内容组织

每个三级标题下的内容遵循以下模式：

1. **定义/概念**
2. **分类/类型**
3. **性质/特征**
4. **判断方法/规则**
5. **实际应用**
6. **注意事项/易错点**
7. **总结表格**

### 4.4 章节安排

- **二级标题划分主要章节**，逻辑清晰。
- **三级标题划分章节内的子主题**。
- **避免使用四级标题**；如果必须使用，确保其内容不会被包含在目录中。

### 4.5 总结与拓展

- **每个重要知识板块后附总结表格**。
- **表格涵盖**「类型-例子-核心特征」「方法-内容-举例」等维度。
- **有「小结」类的三级标题**，用于收束多个子概念。
- **在主要章节结束后，可包含「拓展内容」「总结」或「注意事项」部分**。

### 4.6 考试导向内容

- 明确标注「做题识别要点」「应试要点总结」「核心考点」。
- 提供解题线索和识别模式。
- 提供记忆口诀和快速回忆方法。

### 4.7 补充内容层次

| 层级     | 标签                     | 格式                    |
| -------- | ------------------------ | ----------------------- |
| 基础内容 | 无标签                   | 正文讲述                |
| 拓展内容 | 「补充」「深度拓展」     | `> **补充**` 引用块     |
| 前沿内容 | 「前沿衔接」「大学视野」 | `> **大学视野**` 引用块 |
| 历史背景 | 「科学史话」             | `> **科学史话**` 引用块 |
| 实际应用 | 「前沿应用」             | `> **前沿应用**` 引用块 |

## 五、学科专用规范

### 5.1 化学文档

- 化学反应统一使用 `$\ce{...}$` 语法。
- 反应条件用 `->[\text{条件}]` 标注。
- 重要的反应方程用独立 `$$...$$` 块展示。
- 反应物和产物可用 `\underset{中文名}{化学式}` 标注名称。
- 物质性质框架：物理性质 → 化学性质 → 制备方法 → 应用 → 检验除杂。
- 化学反应需要讲解反应机理。
- 涉及有毒、危险物质时，必须包含安全注意事项。

```md
$$
\ce{H2 + O2 ->[点燃] H2O}
$$
```

```md
<center markdown>
$\ce{H2 + O2 -> H2O}+$ 能量
</center>
```

### 5.2 物理文档

- 物理公式中的单位使用 `$\pu{...}$` 格式。
- 重要的物理公式用独立公式块展示，可用 `\boxed{}` 标记。
- 范围使用 `$10 \sim 20$` 格式。

### 5.3 生物文档

- 公式中的中文描述使用 `<center markdown>` 标签将中文置于公式外部：

```md
<center markdown>
同化量（GPP）$=$ 净初级生产量（NPP）$+$ 呼吸消耗（R）
</center>
```

### 5.4 实验文档

实验文档结构：

1. 反应原理
2. 实验装置（配图）
3. 操作步骤
4. 注意事项
5. 拓展对比

### 5.5 技术文档

技术文档结构：

1. 概述
2. 原理
3. 操作流程
4. 应用案例
5. 常见问题

## 六、SEO 参考指南

**创建实用、可靠且以人为本的内容**

Google 的自动排名系统旨在优先展示旨在为受众带来价值的实用、可靠的信息，而非旨在操纵搜索引擎排名的内容。本页面旨在帮助创作者评估其是否正在制作符合上述标准的内容。

**对您的内容进行自我评估**

根据以下问题对您的内容进行评估，有助于您衡量所制作的内容是否实用且可靠。除了向自己提出这些问题外，建议您邀请您所信任且与您的网站无利益关联的人士为您提供客观真实的评估。

此外，建议您审查曾遭遇的流量或排名下滑情况。哪些页面受到的影响最大？针对的是哪些类型的搜索？请深入分析这些页面，参照本文列出的问题来了解其不足之处。

**内容与质量问题**

- 该内容是否提供了原创的信息、报道、研究或分析？
- 该内容对相关主题的描述是否充实、完整且全面？
- 该内容是否提供了具有深度的见解或并非泛泛而谈的趣味性信息？
- 若内容借鉴了其他来源，其是否避免了单纯的复制或改写，而是提供了实质性的附加价值与原创性？
- 主标题或页面标题是否提供了具有描述性且实用的内容摘要？
- 主标题或页面标题是否避免了采用夸大其词或具有煽动性质的表述？
- 这是您会想要添加书签、与朋友分享或向他人推荐的那类页面吗？
- 您是否期望在印刷版杂志、百科全书或书籍中看到该内容，或被其引用？
- 与搜索结果中的其他页面相比，该内容是否提供了实质性的价值？
- 该内容是否存在任何拼写或文风方面的问题？
- 内容的制作质量是否精良，还是显得粗制滥造或仓促而成？
- 该内容是否由大量创作者批量生产、外包，或散布于庞大的网站网络中，导致单个页面或网站无法获得足够的关注与精细打磨？

**专业性问题**

- 内容呈现信息的方式是否令人信服（例如：清晰的信息来源、相关专业知识的证据、作者背景或发布网站的背景），如提供指向作者页面或网站“关于”页面的链接？
- 如果有人对发布该内容的网站进行调查，他们是否会产生该网站在此类主题上备受信任或被广泛公认为权威机构的印象？
- 该内容是否由在该主题上具有明确认知及专长的专家或资深爱好者撰写或审核？
- 该内容是否存在任何轻易即可证伪的事实性错误？

**提供出色的页面体验**

Google 的核心排名系统旨在奖励能够提供良好页面体验的内容。希望在我们的系统中取得成功的网站所有者不应仅仅关注页面体验的一两个方面。相反，应检查您是否在多个方面均提供了全面而出色的页面体验。如需获取更多建议，请参阅我们的《了解 Google 搜索结果中的页面体验》页面。

**专注于以人为本的内容**

“以人为本的内容”是指主要为受众创建的内容，而非为了操纵搜索引擎排名而创建的内容。您该如何评估自己是否在创作以人为本的内容？如果您对以下问题的回答均为“是”，则说明您在“以人为本”的方向上步入正轨：

- 您的业务或网站是否有现有或潜在的目标受众？如果他们直接访问您的网站，是否会觉得该内容实用？
- 您的内容是否明确展现了第一手专业知识与知识深度（例如：因实际使用过某项产品或服务，或实际游览过某地而获得的专业经验）？
- 您的网站是否具有明确的核心目的或焦点？
- 读者在阅读您的内容后，是否会觉得他们已了解足够的主题信息并有助于实现其目标？
- 读者在阅读您的内容后，是否会觉得获得了一次令人满意的体验？

**避免创建以搜索引擎为中心的内容**

我们建议您专注于创建以人为本的内容，从而在 Google 搜索中取得成功，而不是为了提高搜索引擎排名而专门制作以搜索引擎为中心的内容。如果您对以下部分或全部问题的回答为“是”，这就构成了预警信号，提示您应重新评估您的内容创建方式：

- 该内容是否主要为了吸引来自搜索引擎的访问量而制作？
- 您是否围绕众多不同主题制作了大量内容，以期其中某些内容能在搜索结果中获得良好排名？
- 您是否在大规模使用自动化技术生成涵盖众多主题的内容？
- 您是否主要对他人的观点进行总结，而未添加太多实质性价值？
- 您撰写某些主题的内容是否仅仅因为它们看起来正在流行，而不是因为您的现有受众确实需要？
- 您的内容是否会让读者觉得他们在阅读后仍需继续搜索其他信息？

**了解 Google 搜索结果中的页面体验**

Google 的核心排名系统旨在奖励能够提供良好页面体验的内容。希望在我们的系统中取得成功的网站所有者不应仅仅关注页面体验的一两个方面。相反，应检查您是否在多个方面均提供了全面而出色的页面体验。

**对您的内容页面体验进行自我评估** 如果您对以下问题的回答均为“是”，则说明您在提供良好页面体验方面可能已步入正轨：

- 您的页面是否具备良好的核心网页指标（Core Web Vitals）？
- 您的页面是否以安全的方式提供服务？
- 您的内容在移动设备上是否显示良好？
- 您的内容是否避免了使用过多会分散读者注意力或干扰主要内容的广告？
- 您的页面是否避免了使用侵入式插屏广告？
- 您的页面设计是否便于访问者轻松将主要内容与页面上的其他内容区分开来？

这些问题并未涵盖需要考虑的所有页面体验方面。然而，参考此类问题以及查阅相关资源，将有助于您向提供全面而出色的页面体验这一目标靠拢。

## 七、综合检查清单

在完成文档编写或重构后，使用以下清单进行自检：

### 排版检查

- [ ] 一级标题是否唯一
- [ ] 中英文、中文与数字之间是否加了空格
- [ ] 是否使用全角中文标点
- [ ] 数字是否使用半角字符
- [ ] 公式是否使用正确的格式（行内 `$...$`，独立 `$$...$$`）
- [ ] `$$` 和公式主体是否换行
- [ ] 公式与正文之间是否有空行
- [ ] 公式上一行是否加了引导性标点
- [ ] 列表项之间是否有空行
- [ ] 缩进是否使用四个空格（非 TAB）
- [ ] 图片是否指定了宽度
- [ ] 是否避免使用 `---` 作为分隔线
- [ ] 代码块是否与正文有空行

### 内容检查

- [ ] 是否使用第一人称「我们」
- [ ] 关键术语是否同时标注中英文
- [ ] 是否区分「的」和「地」
- [ ] 是否避免使用「得」字
- [ ] 是否对用户称呼「你」而非「您」
- [ ] 公式内是否尽量避免使用中文
- [ ] 化学式是否使用 `$\ce{...}$`
- [ ] 物理量是否使用 `$\pu{...}$`
- [ ] 范围是否使用 `$10 \sim 20$` 格式
- [ ] 例题是否有先题目后解析的格式

### 结构检查

- [ ] 是否遵循「概念 → 性质 → 应用」框架
- [ ] 二级标题划分是否逻辑清晰
- [ ] 三级标题是否划分具体知识点
- [ ] 是否有总结表格
- [ ] 拓展内容是否使用引用块标记
- [ ] 章节之间是否有前后呼应

### 内容质量自检

以下问题来自 Google 的内容质量指南，可用于自检：

**内容和质量**：

- 内容是否提供了原创的信息、报道、研究或分析？
- 内容是否对主题提供了实质性、完整或全面的描述？
- 内容是否提供了超出显而易见的深刻分析或有趣信息？
- 如果内容借鉴了其他来源，是否避免了简单地复制或重写，而是提供了实质性的附加价值和原创性？
- 主标题或页面标题是否提供了描述性的、有用的内容摘要？
- 主标题或页面标题是否避免了夸张或震惊性质？
- 这是你想要收藏、分享给朋友或推荐的页面吗？
- 你会期望在印刷杂志、百科全书或书籍中看到或引用此内容吗？

**专业性**：

- 内容是否以让你信任的方式呈现信息，如清晰的来源、专业知识的证据、作者或发布网站的背景？
- 如果有人研究该内容的制作网站，是否会留下其值得信赖或被广泛认可的印象？
- 此内容是否由展示出对该主题有深入了解的专家或爱好者撰写或审查？
- 内容是否存在容易验证的事实错误？
