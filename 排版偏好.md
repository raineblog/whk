# 排版与叙述偏好

本文档记录了在撰写数学类教程文档时，用户偏好的排版格式和叙述风格。

## 结构化标签

使用全角方括号 `【】` 作为结构化标签，标注不同类型的内容：

- **【特征识别】**：题型的关键特征和识别方法
- **【基准路径】**：常规解题方法（Standard View）
- **【高维路径】**：高级或巧妙的解题方法（High-Level View）
- **【避坑指南】**：需要注意的陷阱和常见错误（Self-Correction）
- **【核心洞察】**：关键的理论洞察
- **【实用判定方法】**：实用的判定技巧

## 标题格式

使用 `——` 破折号连接标题与说明：

```
【基准路径（Standard View）】—— 连环求导与隐零点代换
【高维路径（High-Level View）】—— 泰勒级数同阶对消
```

使用 `##` 和 `###` 三级标题划分主要章节，尽量避免使用四级标题。

## 术语标注

关键技术术语同时标注中英文：

- **【高维路径（High-Level View）】**
- **【基准路径（Standard View）】**
- **Lipschitz 常数**
- **MVT**（Mean Value Theorem）

## 强调格式

使用 `*...*` 斜体格式强调注意事项或局限性：

> *局限性：计算量大，考试时极易在多阶求导中出现符号错误。*

使用 `**...**` 粗体格式强调关键概念：

> **核心思想**：将双变量差商问题瞬间转化为单变量导数比的最值问题。

## 列表格式

### 列表项之间必须有空行

有序列表和无序列表的任何 item 之间必须有一个空行：

```md
1. 第一步

    详细内容...

2. 第二步

    详细内容...

3. 第三步
```

### 缩进规范

所有缩进必须用四个空格，不能用 TAB。

### 多行公式格式

行内公式使用 `$...，独立公式块使用 `$...$`。

公式与正文之间必须有空行：

```md
根据经典不等式：

$
1+1>2
$

我们得到...
```

必须是三个 ` 符号，必须与正文有空行。如果段落是列表则同步缩进。

公式上一行（上一个段落）应加上引导性标点符号，例如冒号。如果这个段落前面已经有引号了，例如「提示：参考下面的公式。」就需要用句号。如果段落内容与下一个公式比较连贯，可以用末尾的逗号。

## 例题格式

使用加粗标题标注例题：

```
**例 1：普通 MVT 原型（g(x) = x）**
```

## 分隔线

尽量避免使用 `---` 作为分隔线，只在真正需要分隔不同主题时使用。

## 与 AI 写作习惯的差异

以下是我原本习惯但用户不喜欢的写法，需要避免：

| 我原本的习惯 | 用户偏好 |
|------------|---------|
| 较少使用强调 | 使用 `**...**` 强调局限性 |
| 简单列表，无空行 | 列表项之间必须有空行 |
| 使用 TAB 缩进 | 使用四个空格缩进 |
