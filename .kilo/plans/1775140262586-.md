# 构建后处理重构计划

## 背景

当前项目有两个后处理脚本：`gulpfile.js`（旧，Gulp 流式处理）和 `build.ts`（新，Bun 原生）。两者都"能用"但不够完善。MkDocs 使用多个插件（`advanced-seo`、`meta`、`social`、`rss`、`minify_html`）生成 HTML，导致 meta 标签大量重复。需要重构为统一的、深度优化的 `build.ts`。

## 阶段一：环境配置与原始构建

### 1.1 安装构建依赖

```bash
# 安装 uv
curl -LsSf https://astral.sh/uv/install.sh | sh

# 安装 Python 依赖
uv sync --locked --no-dev

# 安装 Pandoc 依赖
uv run --no-sync mkpandocs install-deps

# 安装 Node 依赖
bun install
```

### 1.2 构建原始站点

```bash
# 使用默认 URL 构建（生成 site/ 目录）
uv run --no-sync scripts/generate.py --build --site_url=https://example.com
```

### 1.3 分析原始 HTML 输出

构建完成后分析 `site/` 目录中的 HTML 文件，确认：
- 各插件生成了哪些重复的 meta 标签
- head 标签内的元素排列顺序
- 是否有阻塞渲染的 script 标签
- 结构化数据（JSON-LD）的重复情况

## 阶段二：CSS/JS 深度处理

### 2.1 CSS 处理（替代当前简单 cssnano）

**当前问题**：只用了 `postcss-nested` + `autoprefixer` + `cssnano()`（默认 preset），压缩深度不够。

**改进方案**：
- cssnano 使用 `cssnanoPresetAdvanced`，启用更多压缩规则（合并规则、合并选择器、减少 z-index 等）
- 添加 `postcss-discard-duplicates` 移除重复规则
- 添加 `postcss-merge-rules` 合并相邻选择器
- 添加 `postcss-discard-empty` 移除空规则
- 外部 sourcemap 生成（`.css.map`）
- 明确指定浏览器兼容目标（现代浏览器）

### 2.2 JS 处理（替代当前 Bun.build 方案）

**当前问题**：使用 `Bun.build()` 配合 `format: "iife"` 会把每个 JS 文件包装成 IIFE，可能破坏脚本行为。而且 Bun.build 本质是 bundler 不是 minifier。

**改进方案**：
- 使用 `Bun.Transpiler` API 进行压缩（`minify: true`），这是 Bun 内置的转译器，不会改变模块格式
- 或使用 `terser`（已有依赖）通过 API 直接调用进行深度压缩
- 外部 sourcemap 生成（`.js.map`）
- 配置 `keep_classnames: true, keep_fnames: true` 保留函数名（Material 主题依赖）
- 目标：`ecma: 2020`（现代浏览器）

## 阶段三：HTML 深度优化

### 3.1 问题分析

MkDocs 插件链产生的重复：

| 插件 | 生成的标签 | 重叠风险 |
|------|-----------|---------|
| Material 主题 | charset, viewport, description, title, generator, canonical | 与 advanced-seo 重叠 |
| `meta` 插件 | frontmatter 中的 meta 标签（description, keywords 等） | 与 advanced-seo 重叠 |
| `advanced-seo` 插件 | og:*, twitter:*, JSON-LD, canonical | 与 meta/social 重叠 |
| `social` 插件 | og:image | 与 advanced-seo 重叠 |
| `rss` 插件 | `<link rel="alternate">` | 与自定义 overrides 重叠 |
| 自定义 overrides | fediverse:creator, hreflang, preload, RSS links | 部分重叠 |

### 3.2 优化插件清单

以下插件按执行顺序排列，每个插件独立、可测试：

#### A. `deduplicateMeta` — Meta 标签去重（增强版）

当前 `posthtmlRemoveDuplicates("meta")` 和 `cleanDuplicateSeo()` 都只做基础去重。需要增强：

- **按完整 key 去重**：`name` / `property` / `http-equiv` + `content` 组合
- **优先级策略**：
  - `advanced-seo` 生成的 > `meta` 插件生成的 > Material 主题生成的
  - 判断依据：带有 `data-plugin` 或特定来源标记的优先；无标记的保留最后一个（通常是更精确的）
- **canonical 特殊处理**：`<link rel="canonical">` 优先于 `<meta rel="canonical">`
- **处理 `charset` 和 `viewport`**：只保留第一个
- **处理 `description`**：只保留 content 最长的（通常是 advanced-seo 生成的更完整版本）

#### B. `deduplicateStructuredData` — JSON-LD 去重

- 检测 `<script type="application/ld+json">` 内容
- 按 `@type` + `@id` 或 `url` 去重
- 保留信息量最大的版本

#### C. `deduplicateLinks` — Link 标签去重

- 去重 `<link rel="alternate">`（RSS/Atom/JSON 多个插件可能重复生成）
- 去重 `<link rel="canonical">`（只保留一个）
- 按 `rel` + `href` 组合去重

#### D. `reorderHead` — Head 元素重排序（启用，需修复）

当前被禁用，原因是"可能导致 script 丢失"。修复方案：

**最佳加载顺序**：
1. `charset`（必须是 head 第一个非注释元素）
2. `viewport`
3. `title`
4. 其他 `meta` 标签（按 type 分组）
5. `link[rel=preconnect]` / `link[rel=dns-prefetch]`
6. `link[rel=preload]`
7. `link[rel=stylesheet]`（外部 CSS）
8. `style`（内联 CSS）
9. `script[type=module]`（非阻塞）
10. `script[defer]`（非阻塞）
11. `link[rel=icon]` / `link[rel=alternate]` 等
12. `script[async]`（非阻塞）
13. 其他元素

**安全措施**：
- 不移动带有 `data-no-instant` 或特定标记的脚本
- 保持元素内部相对顺序（稳定排序）
- 保留非标准标签（如 KaTeX 相关）在原位

#### E. `mergeStyleTags` — 合并相邻 Style 标签（启用，需修复）

当前被禁用，原因是"CSS 层叠顺序很重要"。修复方案：

- 只合并 **相邻** 的 `<style>` 标签（中间无其他元素）
- 合并时保持原顺序（保证层叠正确性）
- 不合并带有 `media` 属性的 style 标签（它们有条件加载）
- 不合并带有 `data-*` 特殊标记的 style 标签

#### F. `moveBlockingScripts` — 移动阻塞脚本（启用，需修复）

当前被禁用，原因是"脚本执行时机很重要"。修复方案：

- 只移动 `<head>` 中 **无 `defer`/`async`/`module`** 的 `<script>` 标签
- 移动到 `</body>` 前（不改变执行顺序）
- 不移动包含 `document.write` 的脚本（通过启发式检测）
- 不移动带有 `data-no-move` 属性的脚本

#### G. `minifyInlineCss` — 内联 CSS 压缩

- 对 `<style>` 标签内的 CSS 运行 cssnano 压缩
- 对 `style=""` 属性内的 CSS 运行基础压缩

#### H. `minifyInlineJs` — 内联 JS 压缩

- 对 `<script>` 标签（非 `type="application/ld+json"` 等）内的 JS 运行 terser 压缩
- 保留 `defer`/`async` 属性

#### I. `removeRedundantAttrs` — 移除冗余属性

- `type="text/css"` 在 `<style>` 上（HTML5 默认值）
- `type="text/javascript"` 在 `<script>` 上（HTML5 默认值）
- `language="javascript"`（已废弃）
- 空的 `class=""`、`id=""` 等

#### J. `removeEmptyElements` — 移除空元素

- 空的 `<style></style>`（压缩后可能变空）
- 空的 `<script></script>`（压缩后可能变空）
- 空的 `<meta>` 标签（content 为空）

#### K. `optimizePreload` — 预加载优化

- 确保 `preload` 在 `stylesheet` 之前
- 移除指向同一资源的重复 `preload`/`prefetch`
- 给首屏关键 CSS 添加 `preload`（如果还没有）

### 3.3 自定义 HTML 压缩替代

当前 `minifyHtml()` 使用简单正则，不够可靠。改用 **html-minifier-terser**（已可通过 npm 安装）：

```typescript
import { minify as htmlMinify } from "html-minifier-terser";

const htmlMinOptions = {
  collapseWhitespace: true,
  removeComments: true,
  removeEmptyAttributes: true,
  removeRedundantAttributes: true,
  removeScriptTypeAttributes: true,
  removeStyleLinkTypeAttributes: true,
  minifyCSS: true,
  minifyJS: true,
  useShortDoctype: true,
  continueOnParseError: true,
  // 保留重要属性
  keepClosingSlash: true,
  // 不破坏条件注释
  processConditionalComments: true,
};
```

### 3.4 插件执行顺序

```
PostHTML pipeline:
1. posthtmlImgAlt — 修复空 alt
2. posthtmlPostcss — 内联 CSS 处理
3. posthtmlAltAlways — 确保 alt 存在
4. mergeInlineLonghand — 合并内联 shorthand
5. posthtmlLinkPreload — 预加载优化
6. posthtmlExternalLink — 外部链接处理
7. deduplicateMeta — 增强版 meta 去重
8. deduplicateStructuredData — JSON-LD 去重
9. deduplicateLinks — link 标签去重
10. removeRedundantAttrs — 移除冗余属性
11. mergeStyleTags — 合并相邻 style
12. minifyInlineCss — 内联 CSS 压缩
13. minifyInlineJs — 内联 JS 压缩
14. moveBlockingScripts — 移动阻塞脚本
15. reorderHead — 重排 head 元素
16. removeEmptyElements — 移除空元素
17. optimizePreload — 预加载优化

Then: html-minifier-terser 作为最终 HTML 压缩
```

## 阶段四：build.ts 重写

### 4.1 整体架构

```typescript
// build.ts 结构
import { $, Glob, file } from "bun";
import postcss from "postcss";
import posthtml from "posthtml";
// ... 所有插件导入

// 1. 参数解析
// 2. CSS 处理器配置
// 3. JS 处理器配置
// 4. HTML 插件配置（所有深度优化插件）
// 5. 处理函数：processHtml, processCss, processJs
// 6. 主函数：并行执行 + 统计 + 计时
```

### 4.2 性能优化

- **并行处理**：HTML/CSS/JS 三种文件并行处理（已有）
- **HTML 文件并发**：使用 `Promise.allSettled` 批量处理 HTML 文件（默认并发 16）
- **单文件超时**：每个 HTML 文件处理设置 30s 超时，避免卡死
- **错误处理**：单文件失败不中断整体，收集错误报告
- **进度日志**：每处理 50 个文件输出一次进度

### 4.3 错误处理

```typescript
const results = await Promise.allSettled(tasks);
const failures = results.filter(r => r.status === 'rejected');
if (failures.length > 0) {
  console.error(`${failures.length} 个文件处理失败`);
  // 输出失败文件列表但不中断
}
```
