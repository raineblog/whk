在中文和数字之间添加空格。例如 2025 年、公式 $a+b=c$。

在段落、列表、公式之间添加足够的空行，例如：

- 这是无序列表，与正文段落之间有一个空行。

- 列表之间也要有空行。

- 对于有序列表也这么要求。

行间公式与正文之间也要有空行，特别注意 `$$` 和公式主体之间要有换行，不要在一行写，例如：

$$
E=mc^2
$$

尽量避免公式和列表连在一起写。

数学公式内尽量避免使用中文，如果公式里面有很多中文，可以用下面的方式书写：

<center>
输入 $=$ 输出（有效）$+$ 损耗（无效）
</center>

**请谨记这条要求**，尤其是在生物文档中的公式，通常需要用这种格式书写，以及比如说 $+$ 能量的写法：

<center>
$\ce{H2 + O2 -> H2O}+$ 能量
</center>

<center>
同化量（GPP）$=$ 净初级生产量（NPP）$+$ 呼吸消耗（R）
</center>

同时这意味着 HTML 的块可以写，也要与段落之间有空行，尽量避免使用太多无用的 <p> <a> 标签等，如果能用原生的 Markdown 语法，避免使用 HTML 标签。

如果段落的大部分是中文，那么其中的标点尽可能使用中文全角标点（例如括号、引号等），特别注意“引号的方向”不要反了。另外，**加粗**和正文之间没有空格，*斜体* 和正文（除了全角标点）之间有空格。

如果段落的大部分是英文，那么用英文标点，例如英文段落中对于某个词的中文解释就用英文括号。

特别注意对于数字的数学公式的使用。如果表示的是一个数值（尤其是带有单位的，如果数字非常复杂就用公式）等，尽量使用数学公式，例如质量为 $\pu{1kg}$（物理公式使用 $\pu$ 书写，化学式使用 $\ce$ 书写）、有 $1\%$ 的人、对 $19260817$ 取模、$\pu{9.6E11}$、$\pu{E12}$、$3.14$ 等。对于一些特殊意义的、表示年份的、仅仅是表示数量且是三四位的整数的，尽量不要使用公式，例如 2000 年、5-羟色胺、50 光年、500 年等。另外注意对于范围问题，如果用户的文档用了 $\sim$ 这类的符号，那么就要求使用例如 $30 \sim 50$ 这种语法结构。用户可能的格式例如 10-20 10—20 10～20 10~20 等，你需要都修改为 $10 \sim 20$ 的格式。

注意：请严格按照 4 空格缩进有无序列表等。
